AI Pong

A deterministic C++17 pong simulator rebuilt as an arena for evolutionary self-play — agents train against themselves and a hall-of-fame of their own ancestors, rated on an Elo ladder anchored by scripted bots.

View on GitHub → Leaderboard

What it is

One deterministic simulation core, a swappable agent framework, and two training tracks — GA self-play and PPO — that export to the same model format and compete on one Bradley-Terry Elo league. The strongest trained policy, a memoryless GA champion, beats the perfect-prediction Interceptor bot 95% of the time (Elo 2014 vs. 1843).

Highlights

Play it

An imgui GUI: pick Human, a scripted bot, or any trained model for each paddle — play the champion yourself, or watch two policies rally.

Deterministic core

Pure, IO-free sim with substepped collision — bit-identical for any thread count, so every experiment reproduces exactly.

GA self-play

Evolution against live peers and a hall-of-fame of ancestors, with opponent-diversity fitness and optional 0.5 s memory windows.

PPO track

pybind11 VecPong bindings + CleanRL-style PPO with an adaptive opponent curriculum and function-preserving warm starts.

Elo league

Bradley-Terry ratings pinned to a P-controller baseline and anchored by scripted bots — the apples-to-apples yardstick across both tracks.

Island co-evolution

N architectures co-evolve as separate GA demes under per-island selection, to ask which capacity wins — width vs depth — under co-adaptation.

Live dashboards

imgui training + co-evolution dashboards: margin-vs-bots, the cross-island win matrix, ETA, and pause/stop/eval controls that drive a running job.

Built with

C++17 · Dear ImGui + OpenGL/GLFW · pybind11 + Python PPO · CMake/Ninja · fully hermetic Docker build & test — ./pong build && ./pong test.
